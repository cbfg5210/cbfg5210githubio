抢答app开发介绍

发表于 2016-06-08   |   分类于 编程

道阻且长，行则将至。

本篇博客的目的是给读者们介绍一下一个简单的抢答app的开发过程。本app包括Android端和Bmob云端。
先来看看app运行录像：

开发这个app的流程及要点主要有：
1.抢答规则的制定
2.逻辑流程梳理
3.数据表设计
4.Bmob接口开发
5.原型图设计
6.系统框架的确定
7.Android端功能模块开发

下面就以上提到的几个方面简单介绍一下吧，如果觉得无聊，就想想作者帅气写作的样子吧。

1、抢答规则

制定规则对抢答app的开发而言是首要的，没有定下规则就无法继续下去了，规则包括游戏开始条件、抢题规则、答题规则、计分规则、结束条件等。

规则名称 说明
游戏开始条件 至少两个用户点击开始游戏，服务器随机返回10条题目数据
抢题 有用户抢题成功的话，没抢到题的用户暂时不允许进行抢题和答题操作
答题 抢题成功后限时5秒内答题，超时自动向服务器提交答案
计分 答对一题加一分，答错扣一分，答对答错都进入下一题
游戏结束条件 默认题目有10条，答完即进行分数统计展示，游戏结束

2、逻辑流程梳理

有了游戏规则，接着就是结合规则进行逻辑流程梳理。没有明确逻辑流程的话，在接下来的开发环节中很容易迷路和走错路，难以达到目的。这里使用了processon在线作图工具：

4、Bmob接口开发
这里的Bmob接口开发指的是Bmob云端代码接口的编写，主要的接口有：

一、登录接口—Code_Login

参数：

参数名 必选 类型 说明
deviceId 是 string 设备的imei号

返回示例：

1、用户没注册过：

{
  "code":200,
  "msg":"注册成功",
  "nickName":deviceId
  "objectId":"333f491665"
  "avatar":"http://file.bmob.cn/M03/55/D6/oYYBAFcoj8qAQeYEAAAGucO6NjA856.png"
  }

2、用户注册过了：

{
  "code":200,
  "msg":"该设备已经注册过了",
  "nickName":deviceId
  "objectId":"333f491665"
  "avatar":"http://file.bmob.cn/M03/55/D6/oYYBAFcoj8qAQeYEAAAGucO6NjA856.png"
  }

二、加入房间接口—Code_EnterRoom

参数：

参数名 必选 类型 说明
objectId 是 string 用户id
nickName 是 string 用户昵称
avatar 是 string 用户头像
roomId 是 string 房间id

返回示例：

1、该房间正在进行游戏：

{
  "code":119,
  "msg":"竞答已经开始,建议结束后再加入"
  }

2、该房间没有进行游戏：

{
  "code":200,
  "msg":"成功添加到房间",
  "recordObjectId":"333f491665"
  }

备注：

加入房间成功的话会调用Code_JPush接口发送推送消息给房间内的用户。

三、退出房间接口—Code_ExitRoom

参数名 必选 类型 说明
roomId 是 string 房间id
recordObjectId 是 string 关联的Record数据id
objectId 是 string 用户id
nickName 是 string 用户昵称

返回示例： 

{
  "code":200,
  "msg":"成功退出房间"
  }

四、用户开始游戏接口—Code_IStart

参数：

参数名 必选 类型 说明
objectId 是 string 用户id
nickName 是 string 用户昵称
recordObjectId 是 string 关联的Record数据id
roomId 是 string 房间id

返回示例： 

{
  "code":200,
  "msg":"服务器已经通过你的开始请求,请稍等其他用户"
  }

备注：

开始成功的话会调用Code_JPush接口发送推送消息给房间内的用户；统计点击开始游戏的用户人数，如果人数>=2，发送游戏开始倒计时消息并推送题目信息给房间内用户，房间状态变为游戏中。

五、抢题接口—Code_QiangDa

参数：

参数名 必选 类型 说明
roomId 是 string 房间id
objectId 是 string 用户id
nickName 是 string 用户昵称

返回示例：

1、当前正在答题：

{
  "code":119,
  "msg":"当前正在答题状态,抢题失败"
  }

2、当前没有答题：

{
  "code":200,
  "msg":"正在处理抢题结果..."
  }

备注：

抢题成功的话会调用Code_JPush接口发送抢答成功消息给房间内的用户。

六、答题接口—Code_Answer

参数：

参数名 必选 类型 说明
objectId 是 string 用户id
nickName 是 string 用户昵称
roomId 是 string 房间id
index 是 int 题目序号
answer 是 string 用户答案

返回示例： 

{
  "code":200,
  "msg":"服务器正在审核答案..."
  }

备注：

答案正确加一分，错误扣一分；调用Code_JPush接口推送答题情况给房间内用户。

七、答题超时接口—Code_TimeOut

参数：

参数名 必选 类型 说明
objectId 是 string 用户id
nickName 是 string 用户昵称
roomId 是 string 房间id

备注：

调用Code_Answer接口处理，答案默认为’NO’。

八、游戏结束接口—Code_GameOver

参数：

参数名 必选 类型 说明
roomId 是 string 房间id
recordObjectId 是 string 关联的Record数据id

备注：

调用Code_JPush接口通知房间内用户游戏结束；重置Room表中该房间的数据。

九、获取分数接口—Code_GetScore

参数：

参数名 必选 类型 说明
recordObjectId 是 string 关联的Record数据id

返回示例： 

  {
   "d265e60bf6":{"status":2,"score":1},
   "b83450c66f":{"status":2,"score":5}
  }

十、极光推送接口—Code_JPush

参数：

参数名 必选 类型 说明
sendToFlag 是 string 标识，allcounts：推送给房间内的所有用户，all：推送给app的所有用户
recordObjectId 是 string 关联的Record数据id
params 是 string 需要推送的内容

备注：

调用服务器的PHP接口处理。

十一、极光推送PHP接口

参数1：

参数名 必选 类型 说明
flag 是 string 消息标识，有：NewUserEnter、UserExit、UserStart、DaoJiShi、QiangDaSuccess、AnswerWrong、AnswerRight、GameOver

参数2：

flag=NewUserEnter时：

参数名 必选 类型 说明
alias 是 array 推送目标
nickName 是 string 用户昵称
avatar 是 string 用户头像
objectId 是 string 用户id

flag=UserExit/UserStart时：

参数名 必选 类型 说明
alias 是 array 推送目标
nickName 是 string 用户昵称
objectId 是 string 用户id

flag=DaoJiShi时：

参数名 必选 类型 说明
alias 是 array 推送目标
questions 是 array 题目编号
roomId 是 string 房间id

flag=QiangDaSuccess时：

参数名 必选 类型 说明
alias 是 array 推送目标
nickName 是 string 用户昵称

flag=AnswerWrong/AnswerRight时：

参数名 必选 类型 说明
alias 是 array 推送目标
nickName 是 string 用户昵称
nextNum 是 int 下一道题目编号

flag=GameOver时：

参数名 必选 类型 说明
roomId 是 string 房间id

十二、修改昵称接口—Code_UpdateNick

参数：

参数名 必选 类型 说明
objectId 是 string 用户id
newNick 是 string 新昵称

返回示例：

1、昵称已存在

{
  "code":119,
  "msg":"该昵称已存在"
  }

2、昵称不存在

{
  "code":200,
  "msg":"昵称修改成功"
  }
